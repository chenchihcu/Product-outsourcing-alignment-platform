# 系統架構簡化與排版優化

> ✅ **本計畫已於 2026-06-09 實作完畢（commit `e03e11e` 及後續 CSS/CSS 調整）**

## 目標

1. 從帳號資料模型中移除 `name` 欄位，以部門單位為核發帳號/密碼的基礎，簽名檔作為追溯性用途

### 4. `src/components/SignOff.jsx`

**移除 6 處 `currentUser.name` 自動填入簽名欄位的行為**

| 行號 | 區塊 | 刪除內容 |
|------|------|----------|
| 133-135 | RD - 套用簽章 | `if (!data.basicInfo.signOff?.rdConfirm) { handleSignChange('rdConfirm', currentUser.name); }` |
| 153-155 | RD - 上傳簽章 | `if (!data.basicInfo.signOff?.rdConfirm) { handleSignChange('rdConfirm', currentUser.name); }` |
| 236-238 | PE - 套用簽章 | `if (!data.basicInfo.signOff?.engineeringReview) { handleSignChange('engineeringReview', currentUser.name); }` |
| 256-258 | PE - 上傳簽章 | `if (!data.basicInfo.signOff?.engineeringReview) { handleSignChange('engineeringReview', currentUser.name); }` |
| 345-347 | QA - 套用簽章 | `if (!data.basicInfo.signOff?.qaConfirm) { handleSignChange('qaConfirm', currentUser.name); }` |
| 365-367 | QA - 上傳簽章 | `if (!data.basicInfo.signOff?.qaConfirm) { handleSignChange('qaConfirm', currentUser.name); }` |

**改動方式：** 刪除整個 `if` 區塊（保留前後的 `}` 或 `onChange` callback 結構）

例如改前：
```jsx
onClick={() => {
  handleSignChange('rdSignature', currentUser.signature);
  if (!data.basicInfo.signOff?.rdConfirm) {
    handleSignChange('rdConfirm', currentUser.name);
  }
}}
```
改後：
```jsx
onClick={() => {
  handleSignChange('rdSignature', currentUser.signature);
}}
```
